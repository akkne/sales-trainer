## 1. Онбординг

**Роут:** `/onboarding`

**Когда:** После регистрации, один раз.

**Шаги:**
1. **Тип продаж** — выбор из карточек (B2B SaaS, розница, недвижимость, финансы, B2C)
2. **Уровень опыта** — выбор из карточек (новичок, опытный, руководитель)
3. **Выбор навыков** — тогглы для выбора навыков, которые хочет изучать. `Основы продаж` всегда выбран и недоступен для отключения (серый тоггл)

**UI-паттерны:**
- Полноэкранные шаги с прогресс-индикатором сверху
- Крупные карточки для выбора (одиночный выбор на 1-2 шаге)
- Тогглы для мультивыбора на 3 шаге
- Кнопка "Продолжить" внизу

## 2. Дерево навыков (главный экран)

**Роут:** `/tree`

**Назначение:** Главный хаб приложения. Выбор навыка и просмотр прогресса.

**Структура:**
- **Левый сайдбар** — список подписанных навыков с мини-прогрессбарами. Клик = выбор активного навыка
- **Центральная зона** — вертикальный список уроков выбранного навыка (как в Duolingo). Каждый урок — кружок:
	- Зеленый с галочкой = пройден
	- Зеленый с номером = доступен
	- Серый с замком = заблокирован
- **Верхняя панель** — текущий стрик, XP за сегодня, место в лиге

**Интеракции:**
- Клик по доступному уроку → открывает попап с кнопкой "Начать урок" и ссылкой на карту курса
- Клик по заблокированному → показывает "Пройди предыдущий урок"

## 3. Карта курса (внутри навыка)

**Роут:** `/skill/[slug]/map`

**Назначение:** Детальный обзор всех уроков навыка.

**Структура:**
- **Хедер-карточка** — иконка навыка, название, "X из N уроков пройдено", большой процент справа, прогрессбар
- **Список уроков** — карточки с:
	- Кружок-статус (зеленая галочка / зеленый номер / серый замок)
	- Бейдж "Урок N"
	- Бейдж "+N XP"
	- Время "~N мин" (если указано)
	- Название урока (жирное)
	- Описание (2 строки, обрезается)
	- CTA кнопка "Начать урок" для доступных
	- Подсказка "Пройди предыдущий урок" для заблокированных

**Переходы:**
- Вход: из попапа на `/tree` или из хедера `/skill/[id]`
- Клик по CTA → сессия урока

## 4. Сессия урока (упражнения)

**Роут:** `/session/[lessonId]`

**Назначение:** Последовательное прохождение упражнений урока.

### Типы упражнений

**Multiple Choice (выбери лучший ответ):**
- Ситуация сверху (реплика клиента)
- 2-4 варианта ответа снизу (карточки)
- Выбор → кнопка "Проверить" становится активной
- После ответа: зеленая/красная подсветка правильного варианта, объяснение

**Fill-in-the-Blank (заверши диалог):**
- Диалог с пропуском
- 2-4 варианта для заполнения пропуска
- Аналогичная механика выбора и фидбека

**Free Text (свободный ответ):**
- Ситуация сверху
- Текстовое поле для ввода
- Кнопка "Отправить"
- AI оценивает и показывает развернутый фидбек с баллами

### Общие UI-элементы
- **Прогрессбар** вверху (сколько упражнений пройдено)
- **Кнопка "X"** — выход из сессии (с подтверждением)
- **Хинт для клавиатуры** (только десктоп): "1–4 выбрать · Enter — проверить"
- После ответа: кнопка "Продолжить" (Enter/Space)

### Экран завершения
- Если есть следующий урок: кнопка "Следующий урок →" + название следующего урока
- Если последний урок в навыке: "Все уроки пройдены! 🎉"
- Ссылка "Вернуться к пути" → `/tree`

## 5. AI-диалог (текстовая практика)

**Роуты:**
- `/dialog` — выбор "бандла" (группы сценариев)
- `/dialog/[bundleId]` — выбор режима (сценария)
- `/dialog/[bundleId]/[modeId]` — чат

**Назначение:** Практика продаж в реалистичных диалогах с AI.

### Структура `/dialog`
- Сетка карточек-бандлов (иконка, название, описание)
- Каждый бандл привязан к навыку

### Структура `/dialog/[bundleId]`
- Сетка карточек-режимов (название, описание)
- Пример: бандл "Холодные звонки" → режимы "Обход секретаря", "Работа с возражениями"

### Экран чата `/dialog/[bundleId]/[modeId]`
**Лейаут:**
- **Левый сайдбар** (сворачиваемый) — история сессий
	- "Новый диалог" кнопка сверху
	- Сессии сгруппированы по дате (Сегодня, Вчера, N дн. назад)
	- Каждая сессия: название режима, кол-во сообщений, заработанный XP
- **Центральная зона** — чат
	- Сообщения пользователя: зеленые, справа
	- Сообщения AI: серые, слева
	- Индикатор печати AI
	- Поле ввода снизу

**Завершение диалога:**
- AI сигнализирует конец тегом `[DIALOG_END]` (скрыт от пользователя)
- Появляется кнопка "Завершить диалог"
- Модалка с фидбеком: оценка, советы, заработанный XP (0-100)

## 6. Голосовая практика

**Роут:** тот же `/dialog/[bundleId]/[modeId]`, если режим поддерживает голос

**Назначение:** Практика продаж голосом (как реальный звонок).

**UI:**
- Центральная кнопка микрофона (круглая)
- Состояния кнопки:
	- **Idle** — серая иконка микрофона, тап для старта
	- **Listening** — зеленое пульсирующее кольцо вокруг, VAD активен
	- **Processing** — индикатор загрузки, ждем AI
	- **Playing** — иконка динамика, AI говорит
	- **Disabled** — серая, недоступна (AI отвечает или нет конфига)

**Поток:**
1. Пользователь говорит → зеленое кольцо
2. Пауза 600мс → распознавание заканчивается
3. Ответ AI генерируется → аудио воспроизводится
4. Микрофон снова доступен

**Текстовый режим vs голосовой:** Режим определяется настройкой `voiceEnabled` на режиме. Переключение mid-session невозможно.

## 7. Справочник продаж

**Роут:** `/guidebook`

**Назначение:** Теоретическая база по техникам продаж.

**Структура:**
- **Поиск** — real-time фильтрация по названию и контенту
- **Чипы категорий** — "Все", "Возражения", "Холодные звонки", "Закрытие сделки" и т.д.
- **Карточки техник:**
	- Свернутое состояние: бейдж категории, теги-пилюли, название, 2 строки превью
	- Развернутое состояние: полный markdown-контент, ссылка на связанный навык
	- Только одна карточка раскрыта одновременно

**Пустое состояние:** Показывается когда нет материалов по фильтрам.

## 8. Профиль

**Роут:** `/profile`

**Секции:**
- **Статистика:** стрик (дни подряд), общий XP, пройденные навыки
- **Мои навыки:** тогглы для управления подпиской на навыки
	- `Основы продаж` — всегда включен (серый тоггл)
	- Другие навыки — включить/выключить
	- Отключение = навык исчезает из `/tree`, но прогресс сохраняется

---

## 9. Лиги и рейтинг

**Роут:** `/league`

**Назначение:** Еженедельное соревнование между пользователями.

**UI:**
- Название текущей лиги (Bronze, Silver, Gold, Diamond)
- Список из ~30 участников: аватар, имя, XP за неделю
- Текущий пользователь подсвечен
- Индикаторы зон:
	- Зеленая зона (топ-10) — повышение в следующую лигу
	- Красная зона (нижние 5) — понижение
- Таймер до конца недели

---

## 10. Достижения

**Механика:** При разблокировке достижения во время сессии показывается тост.

**Тост:**
- Появляется сверху, зеленая рамка
- Содержимое: эмодзи иконка, "Достижение разблокировано!", название, описание
- Автоскрытие через 4 секунды
- Клик = мгновенное скрытие
- При множественных достижениях — очередь (показываются по одному)

## 11. Админ-панель

**Роуты:** `/admin/*`

**Доступ:** Только для Admin и SuperAdmin ролей.

### Навигация (сайдбар):
- Навыки
- Диалоги
- Справочник
- Пользователи (только SuperAdmin)

### Навыки `/admin/skills`
- Таблица: название, slug, иконка, порядок
- CRUD: создание, редактирование, удаление
- Drill-down в уроки навыка

### Уроки `/admin/skills/[id]`
- Таблица уроков навыка
- Drill-down в упражнения урока

### Упражнения `/admin/skills/[id]/lessons/[lessonId]`
- Таблица упражнений
- Редактор JSON-контента для каждого типа упражнения

### Диалоги `/admin/dialog`
- Таблица бандлов
- CRUD бандлов
- Drill-down в режимы бандла

### Режимы диалогов `/admin/dialog/[bundleId]`
- Таблица режимов
- Редакторы системных промптов (для чата и для фидбека)
- Тоггл голосового режима + поле Voice ID

### Справочник `/admin/reference`
- Глобальная таблица материалов (все навыки)
- CRUD: категория, теги, markdown-контент, привязка к навыку

### Пользователи `/admin/users` (SuperAdmin)
- Таблица пользователей: email, имя, роль
- Изменение роли (User/Admin/SuperAdmin)
- Бейджи ролей: серый=User, синий=Admin, фиолетовый=SuperAdmin

**UI-принципы админки:**
- Минимализм, никаких анимаций
- Стандартные формы и таблицы
- Модалки для создания/редактирования
- Inline подтверждение удаления (кнопка меняется на "Подтвердить?")

## Общие UI-компоненты

### Нижняя навигация (мобильная)
- Дерево навыков
- Диалог
- Справочник
- Профиль

### Клавиатурные шорткаты (десктоп)
- `1-4` — выбор варианта в упражнениях
- `Enter/Space` — отправить ответ / продолжить

### Цветовая схема
- Зеленый — основной акцент, успех, активные элементы
- Серый — неактивные, заблокированные элементы
- Красный — ошибки, неправильные ответы

### Анимации
- Только для значимых переходов: правильный/неправильный ответ, прогресс, разблокировка
- Пульсация зеленого кольца для голосового режима

---

## Технические детали для дизайнера

- **Адаптивность:** Mobile-first, десктоп = расширенный вариант
- **Шрифты:** Tailwind defaults (system fonts)
- **Иконки:** Lucide React
- **Компоненты:** Headless UI + Tailwind
