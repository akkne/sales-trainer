# Скилы и упражнения — справочник для сотрудников

## Часть 1 — Скилы (Skills)

Скил — это раздел обучения (например, «Холодные звонки», «Работа с возражениями»).
Скилы образуют дерево навыков, которое пользователь проходит последовательно.

### Поля скила

| Поле | Тип | Описание |
|------|-----|----------|
| `title` | строка | Название скила, которое видит пользователь. Например: `"Холодные звонки"` |
| `slug` | строка | Короткий уникальный идентификатор в URL-формате. Например: `"cold-calls"`. Используется в URL запросов: `GET /skills/{slug}/lessons` |
| `iconName` | строка | Название иконки для отображения в интерфейсе. Например: `"phone"` или `"handshake"` |
| `sortOrder` | число | Порядок отображения скила в списке. Меньшее число — выше. Например: `1`, `2`, `3` |
| `prerequisiteSkillId` | GUID или null | ID скила, который нужно завершить, чтобы этот разблокировался. Если `null` — скил доступен сразу |
| `applicableSalesTypes` | массив строк | Типы продаж, для которых этот скил актуален. Например: `["b2b", "b2c"]` |

### Как работает slug

Slug — это «адрес» скила в системе. Когда фронтенд хочет получить уроки по скилу, он делает запрос:

```
GET /skills/cold-calls/lessons
```

Именно поэтому slug должен быть уникальным и не меняться после создания — иначе сломаются ссылки.

### Как работает порядок разблокировки

1. Скил с `prerequisiteSkillId = null` — **доступен сразу** при регистрации
2. Скил с `prerequisiteSkillId = <id другого скила>` — **заблокирован** до тех пор, пока пользователь не завершит все уроки в скиле-предшественнике
3. После завершения скила система автоматически ищет следующий заблокированный скил с этим `prerequisiteSkillId` и переводит его в статус `"available"`

### Статусы скила у пользователя

| Статус | Значение |
|--------|----------|
| `locked` | Недоступен — не пройден prerequisite |
| `available` | Доступен для прохождения |
| `in_progress` | Начат (есть хотя бы один пройденный урок) |
| `completed` | Все уроки завершены |

## Часть 2 — Упражнения (Exercises)

Упражнение — это задание внутри урока. В каждом уроке может быть несколько упражнений.

### Поля упражнения

| Поле | Тип | Описание |
|------|-----|----------|
| `lessonId` | GUID | К какому уроку относится упражнение |
| `type` | строка | Тип упражнения (см. ниже) |
| `sortOrder` | число | Порядок упражнения внутри урока. Меньшее — раньше |
| `content` | JSON-объект | Содержимое упражнения. Структура зависит от типа |

### Типы упражнений

#### `multiple_choice` — выбор из вариантов

Пользователь выбирает один правильный вариант из нескольких.

**Структура content:**
```json
{
  "question": "Что делать при возражении 'Дорого'?",
  "options": ["Снизить цену", "Выяснить ценность", "Завершить разговор", "Предложить рассрочку"],
  "correctOptionIndex": 1,
  "explanation": "Нужно выяснить, что именно кажется дорогим — цена или ценность"
}
```

**Проверка:** сравнивается `selectedOptionIndex` из ответа пользователя с `correctOptionIndex` из контента. Если совпадают — правильно (score = 100), иначе неправильно (score = 0).

---

#### `fill_blank` — вставить пропущенное слово

Пользователь выбирает правильный вариант для заполнения пропуска в тексте.

**Структура content:** аналогична `multiple_choice` — те же поля `correctOptionIndex` и `options`.

**Проверка:** такая же, как у `multiple_choice` — по индексу выбранного варианта.

---

#### `free_text` — свободный ответ (оценивается ИИ)

Пользователь пишет развёрнутый ответ. Ответ оценивает OpenAI GPT-4o-mini.

**Структура content:**
```json
{
  "situation": "Клиент говорит: 'Мне нужно подумать'. Как ты отвечаешь?",
  "evaluationCriteria": "Менеджер должен выяснить реальное возражение, не давить, предложить помощь"
}
```

**Проверка:** запрос уходит в OpenAI, который возвращает:
- `score` — от 0 до 100
- `feedback` — текстовый фидбек на русском
- `isCorrect` — true если score ≥ 60

Если ключ OpenAI не настроен — упражнение считается пройденным с score = 70.

---

### Что происходит после правильного ответа

1. Записывается попытка (`UserExerciseAttempt`) с результатом
2. Пользователю начисляется XP (количество берётся из поля `xpReward` урока)
3. Урок помечается как `"completed"`
4. Обновляется счётчик пройденных уроков в скиле
5. Если все уроки скила пройдены — скил закрывается как завершённый, следующий скил разблокируется
6. Обновляется streak (серия дней обучения подряд)

## API — краткая шпаргалка

| Метод | URL | Что делает |
|-------|-----|------------|
| `GET` | `/skills/{slug}/lessons` | Список уроков скила с прогрессом пользователя |
| `GET` | `/lessons/{id}/exercises` | Список упражнений урока |
| `POST` | `/exercises/{id}/submit` | Отправить ответ на упражнение |
| `GET` | `admin/skills` | Все скилы (только админ) |
| `POST` | `admin/skills` | Создать скил (только админ) |
| `PUT` | `admin/skills/{id}` | Изменить скил (только админ) |
| `DELETE` | `admin/skills/{id}` | Удалить скил (только админ) |
| `POST` | `admin/lessons/{id}/exercises` | Создать упражнение (только админ) |
| `PUT` | `admin/exercises/{id}` | Изменить упражнение (только админ) |
| `DELETE` | `admin/exercises/{id}` | Удалить упражнение (только админ) |
