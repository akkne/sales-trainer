# CODESTYLE.md

## Это закон. Без исключений.

---

## Именование

**Без сокращений. Никогда.** Название объясняет назначение без контекста.

- Переменные и поля — длинные, описательные: `currentAuthenticatedUser`, `totalCompletedExerciseCount`
- Булевы — всегда с префиксом `is`, `has`, `can`, `should`, `was`
- Коллекции — множественное число с указанием содержимого: `unlockedSkillNodes`, `weeklyLeagueScoresByUserId`
- Методы — глагол + существительное полностью: `ProcessExerciseSubmission`, `FindUserByIdentifier`
- Классы — полное название роли: `ExerciseEvaluationService`, не `ExSvc`
- Интерфейсы — префикс `I` + полное название: `IExerciseRepository`

---

## Структура файлов

**Один класс — один файл. Всегда.**

Файл называется точно как класс внутри. Никаких двух классов, record+interface, dto+validator в одном файле.

Организация по фичам, не по типам:
```
Features/Exercises/ExerciseController.cs
Features/Exercises/ExerciseEvaluationService.cs
Features/Exercises/ExerciseRepository.cs
Features/Exercises/ExerciseSubmissionDto.cs
```

---

## Паттерны — обязательные

**Service Layer** — вся бизнес-логика в сервисах. Контроллер только принимает запрос, вызывает сервис, возвращает результат.

**Strategy** — разные типы упражнений (multiple choice, free text, voice) реализуются как отдельные стратегии с общим интерфейсом.

**Factory** — создание сложных объектов (дерево навыков, персонализированный онбординг) через фабрики.

**Observer** — события после действий пользователя (завершил упражнение → обновить стрик, обновить лигу, начислить очки) через наблюдателей.

**Custom Hook** (фронтенд) — вся логика вынесена из компонентов в хуки. Компонент только рендерит.
