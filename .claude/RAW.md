# **СТЕК**

>
>
>
> ## Архитектура
>
> Модульный монолит. Один бэкенд, один фронтенд, три базы данных.
>
> `Next.js (TypeScript) → HTTP REST (JSON) → ASP.NET Core (C#)
>                                               ├── PostgreSQL (основная БД)
>                                               ├── MongoDB (чат, диалоги)
>                                               ├── Redis (кэш, сессии, лидерборды)
>                                               └── OpenAI API (оценка свободных ответов)`
>
> Аутентификация через JWT (access + refresh). Фоновые задачи через Hangfire.
>
> ---
>
> ## Frontend
>
> **Стек:** Next.js, TypeScript, Tailwind CSS, Zustand (клиентский стейт), React Query (серверный стейт), Framer Motion (анимации).
>
> **Роутинг (App Router):**
>
> - `(auth)/` — логин, регистрация, онбординг
> - `(main)/tree` — главный экран, дерево навыков
> - `(main)/skill/[id]` — уроки внутри навыка
> - `(main)/exercise/[id]` — экран упражнения
> - `(main)/reference/[id]` — справочник по навыку
> - `(main)/league` — рейтинг / лига
> - `(main)/profile` — профиль и статистика
>
> **Правила:**
>
> - Все запросы к API через единый fetch-wrapper с автоподстановкой JWT и обработкой 401.
> - Типы на фронте зеркалят DTO бэкенда. При изменении DTO — обновлять типы.
> - Компоненты упражнений принимают единый интерфейс и рендерят UI в зависимости от `exercise.type`.
> - Анимации только для значимых переходов (ответ правильный/неправильный, прогресс, разблокировка).
>
> ---
>
> ## Backend
>
> **Стек:** ASP.NET Core Web API, C#, Entity Framework Core (Postgres), MongoDB.Driver, StackExchange.Redis, Hangfire, FluentValidation, Serilog, Swashbuckle (Swagger).
>
> **Структура — вертикальная нарезка по фичам.** Каждая фича — папка с контроллером, сервисом и DTO. Не делать горизонтальные слои (Controllers/, Services/, Repositories/).
>
> **Фичи:** Auth, Onboarding, SkillTree, Lessons, Exercises, Reference, Gamification, League, Profile.
>
> **Правила:**
>
> - Нет репозиториев-обёрток — EF Core DbContext используется напрямую в сервисах.
> - DTO ≠ Entity. Никогда не возвращать EF Core сущности из контроллеров.
> - Все вызовы OpenAI проходят через единый `AiEvaluationService` с rate limiting, таймаутами и логированием.
> - Async/await везде. Nullable reference types включены.
> - HTTP-коды: 200, 201, 400 (валидация), 401, 404, 500 (глобальный middleware).
>
> ---
>
> ## Базы данных
>
> ### PostgreSQL — основная база
>
> Все структурированные данные: пользователи, профили, навыки, уроки, упражнения, прогресс, стрики, XP, лиги, справочные материалы.
>
> **Ключевые таблицы:**
>
> - `users` — id, email, password_hash, display_name
> - `user_profiles` — sales_type (b2b_saas / retail / real_estate / finance / b2c), experience_level, goal
> - `skills` — slug, title, sort_order, prerequisite_skill_id (для последовательной разблокировки), sales_types[]
> - `lessons` — привязан к skill, sort_order, difficulty_level, xp_reward
> - `exercises` — привязан к lesson, type (multiple_choice / fill_blank / free_text), content (jsonb — структура зависит от типа)
> - `user_skill_progress` — status (locked / available / in_progress / completed), счётчики пройденных уроков
> - `user_lesson_progress` — status, best_score
> - `user_exercise_attempts` — ответ пользователя (jsonb), is_correct, score, ai_feedback (jsonb)
> - `user_streaks` — current_streak, longest_streak, last_activity_date
> - `user_xp` — amount, source (exercise / streak_bonus / league_bonus), earned_at
> - `leagues` — tier (bronze / silver / gold / diamond), week_start, week_end
> - `league_memberships` — weekly_xp, rank, promoted/demoted
> - `reference_materials` — привязан к skill, title, content (markdown)
>
> ### MongoDB — чат и неструктурированные данные
>
> На этапе MVP используется минимально. Одна коллекция `chat_messages`: user_id, exercise_id, role (user / ai_character / system), character_slug, content, metadata, created_at.
>
> В будущем: `call_transcripts` для хранения транскриптов звонков и scorecards.

# **1. Онбординг (одноразовый, при регистрации)**

Регистрация через email или Google. После регистрации — 3-4 экрана: выбор типа продаж (B2B SaaS, розница, недвижимость и т.д.), выбор уровня опыта (новичок, опытный, руководитель), выбор цели (хочу научиться закрывать сделки, хочу улучшить холодные звонки, хочу всё). На основе ответов формируется персональное дерево навыков. Не нужно делать это сложным — достаточно трёх шагов.

# **2. Главный экран — дерево навыков**

Это аналог главной страницы Duolingo со скриншота, где модули идут сверху вниз. У нас вместо "Модуль 1, Модуль 2" будут навыки: "Холодные звонки", "Квалификация", "Возражения" и т.д. Каждый навык показывает прогресс (как у Duolingo — процент и галочка "пройдено"). Навыки разблокируются последовательно — нельзя перейти к возражениям, не пройдя квалификацию. На этом же экране справа (или сверху на мобилке): текущий стрик, очки за сегодня, место в лиге.

# **3. Экран внутри навыка — список уроков**

Как на скриншоте с зелёными кружками и сундуками. Внутри навыка "Работа с возражениями" будет 5-7 уроков, идущих от простого к сложному. Каждый урок — это кружок на карте. Первые уроки — тесты и пропуски, последние — свободный текст. Плюс "Справочник" — кнопка, которая открывает теорию по навыку (как на скриншоте с ключевыми фразами, только у нас это будут ключевые техники и скрипты).

# **4. Экран упражнения — сам урок**

Для MVP достаточно трёх типов. Первый — "выбери лучший ответ": сверху ситуация (клиент говорит что-то), снизу 2-3 варианта ответа, один правильный. Точно как на скриншоте Duolingo с "She wants to know about..." — только у нас будет "Клиент: У нас уже есть решение. Лучший ответ:". Второй — "заверши диалог": как на скриншоте с "Завершите диалог", где персонаж говорит фразу и нужно выбрать ответ. У нас аналогично — "Скептик Сергей" говорит реплику, ты выбираешь свой ответ. Третий — свободный ответ текстом: поле ввода, пользователь пишет ответ, ИИ оценивает и даёт фидбек.

После каждого ответа — экран результата: зелёная плашка "Отлично!" если правильно, красная с объяснением если нет. Прогресс-бар вверху, как у Duolingo.

# **5. Экран справочника**

Как на скриншоте с ключевыми фразами. У нас вместо фраз — ключевые техники для каждого навыка. Например, для "Работы с возражениями": техника "Согласись и переведи", техника "Вопрос-уточнение", техника "Социальное доказательство". Каждая техника — короткое описание плюс пример диалога. Это теоретическая база, к которой пользователь возвращается.

# **6. Экран профиля и статистики**

Стрик (количество дней подряд), общее количество очков, пройденные навыки, средняя оценка по типам упражнений. Для MVP — простая страница без сложных графиков.

# **7. Экран лиги (рейтинг)**

Еженедельный рейтинг: список из 30 участников, отсортированных по очкам за неделю. Имя, аватар, очки. Топ-10 переходят в следующую лигу, нижние 5 вылетают. Для MVP можно начать с одной лиги и простого списка.
